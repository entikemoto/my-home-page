医師がメスの手入れを怠らないように、AIを使って仕事をする人間は道具のメンテナンスを怠ってはいけない。

私は現在、Claude Code を司令塔として、Cursor・Gemini CLI・Codex CLI を目的に応じて使い分けるエージェント連携の開発環境を運用している。この構成は3ヶ月前とは大きく変わっているし、おそらく3ヶ月後もまた変わっているだろう。

## なぜ毎日アップデートするのか

単純な話で、AIツールの進歩速度が人間の慣れの速度より速いからだ。

Claude Code が登場した当初、私はそれを「便利なコード補完ツール」として使っていた。ところが数ヶ月で、設計の相談相手・タスク管理ツール・記事生成パイプラインの司令塔と、役割が根本から変わった。同じツールを「先月と同じ使い方」で使い続けることは、性能を引き出せていないだけでなく、機会コストを失っていることになる。

## 医師という職業との相性

医師は本来、「最新のエビデンスを実臨床に落とす」ことに慣れている職業だ。毎朝のジャーナルチェック、学会での知識のアップデート、プロトコル改訂への対応——これは全部「自分のやり方をアップデートし続ける」行為だ。

AIツールへの適応も同じだと思っている。新しいバージョンのリリースノートを読み、実際に使ってみて、自分のワークフローに取り込む。これは特別なことではなく、プロフェッショナルとして当然の行為だ。

## 現在の構成

私のAgentic Dev OSは以下の思想で設計している。

- **Claude Code（Sonnet）** が司令塔。設計・実装・タスク管理を担う
- **Gemini CLI** に複数ソースの調査や大量ドキュメント読解を委譲する
- **Codex CLI** にコードレビューとセキュリティ監査を任せる
- 重要な設計判断は **Opus** にエスカレーションし、迷いが残れば **GPT-5.4** にセカンドオピニオンを求める

この構成はCLAUDE.mdというファイルに明文化されており、Claude Code はそれを毎セッション参照する。いわば「AIとの作業規約」だ。

## アップデートのコスト

「毎日更新するのは大変では？」と思われるかもしれない。実際には、一度仕組みを整えると維持コストは思ったより低い。

変更の大半は小さな調整だ。委譲ルールの一行変更、新しいモデルへの切り替え、ツール間の優先順位の調整。それが積み重なって、3ヶ月後には別物のような効率になっている。

医療でも、毎日の小さな習慣改善が5年後の診療の質に繋がる。同じことだと思っている。
