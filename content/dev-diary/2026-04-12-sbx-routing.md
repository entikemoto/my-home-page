## 何をしたか

Claude Code がコマンドを実行するとき、「自分のMacで直接やる」か「隔離された箱の中でやる」かを自動判断するルールを設定した。

---

## なぜ必要か――非エンジニアへの説明

### AI は「手」を持っている

Claude Code はただ答えを返すだけでなく、実際にファイルを書いたりコマンドを実行したりできる。これは非常に便利だが、同時に「AIが自分のPCで自由に作業できる状態」でもある。

### 「安全な作業」と「危険な作業」がある

たとえば：

- **安全な例**: 記事のファイルを編集する、テストを実行する  
  → 何か間違えても、`git` で元に戻せる

- **危険な例**: インターネットからプログラムをダウンロードして実行する  
  → 悪意あるコードが混じっていても、実行されてしまうと手遅れになる場合がある

これは現実の作業で言えば、「ノートに書く」と「正体不明の薬を飲む」くらいリスクが違う。

### 「サンドボックス」とは何か

サンドボックス（SBX）とは「砂場」のことで、転じて「何をやっても外に影響が出ない隔離された実行環境」を指す。

Dockerというツールを使うと、Mac の中に「使い捨ての仮想マシン」を一瞬で作れる。その中でリスクのある操作を実行すれば、終わったら箱ごと捨てられる。自分のMacは汚染されない。

```
Mac本体（大切な資産）
└── Docker コンテナ（使い捨ての箱）
        └── ここで危険な操作を実行 → 箱ごと廃棄
```

---

## 実装した内容

`CLAUDE.md`（Claude Code が毎セッション読む「作業規約ファイル」）に判定テーブルを追加した。

| 操作 | 環境 |
|------|------|
| ファイル編集・テスト実行・git 操作 | ローカル（Mac本体） |
| 新しいパッケージのインストール | **SBX（Docker）** |
| 外部サービスへの書き込み | **SBX（Docker）** |
| 元に戻せない削除操作 | **SBX（Docker）** |

「迷ったらSBXを選ぶ」をデフォルトにした。

---

## 気づいたこと

このルールで重要なのは「AIが判断する」という点だ。以前は都度確認が必要だったが、CLAUDE.md にテーブルを書いておくだけで、Claude Code が自分でリスクを評価して環境を選ぶようになった。

「AIに自由にやらせる」と「AIを安全に動かす」は矛盾しない。ルールを明文化することで、両立できる。
