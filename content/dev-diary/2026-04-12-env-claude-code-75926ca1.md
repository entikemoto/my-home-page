### この回の要点（先に読む）

- **変わったこと**: Claude Code がコマンドを実行するとき、「Mac本体で直接やる」か「隔離された箱（Docker）の中でやる」かを、作業内容に応じて自動で選ぶようになった。
- **いままでとの違い**: 以前は区別なくMac本体で実行していた。どんな操作でも同じ環境で動いていた。
- **メリット**: リスクのある操作（知らないプログラムのインストールなど）が隔離環境で実行されるため、Mac本体が汚染されない。

### 何が変わったか

#### 🏗️ AIが「安全な操作」と「危険な操作」を区別するようになった

Claude Code はファイルを編集するだけでなく、ターミナルでコマンドを実行できます。これは便利ですが、操作によってはリスクが異なります。

**安全な操作**（Mac本体で直接実行）:
- ファイルの編集・読み込み
- `npm run dev` / `npm run test` などの開発コマンド
- `git` 操作

**リスクのある操作**（Docker の隔離環境で実行）:
- `npm install <パッケージ>` — インターネットからプログラムをダウンロードする
- 外部サービスへの書き込み
- `rm -rf` などの削除操作
- 正体不明のスクリプトの実行

#### 📦 サンドボックス（Docker）とは何か

「砂場」の意味で、何をやっても外に影響が出ない使い捨ての実行環境です。

```
Mac本体（大切な資産）
└── Docker コンテナ（使い捨ての箱）
        └── ここで危険な操作を実行 → 箱ごと廃棄
```

終わったら箱ごと捨てるため、Mac本体は汚染されません。

#### 📋 CLAUDE.md にルールを書いて自動化した

「迷ったらサンドボックスを選ぶ」を原則として、操作カテゴリごとの判定テーブルを `CLAUDE.md`（Claude Code が毎セッション読む作業規約ファイル）に追記しました。これにより、毎回口頭で指示しなくても Claude Code が自動で環境を選ぶようになりました。

### 今回対応したこと

- ✅ `CLAUDE.md` に実行環境ルーティング（ローカル vs SBX）の判定テーブルを追加
- ✅ Docker コマンドの実行形式を明記（`docker run --rm -v $(pwd):/app ...`）
- ✅ 「迷ったら SBX を選ぶ」をデフォルトルールとして設定

> **補足（MyHomePage プロジェクトの場合）**: 日常操作の大半（ファイル編集・ビルド・テスト・git など）はローカルで完結するため、実際に SBX が起動するのは `npm install` 時・外部 API への書き込み・大量削除など例外的なケースのみです。「迷ったら SBX」は分類外の操作が来たときの安全弁であり、日常の作業負担はほぼゼロです。
