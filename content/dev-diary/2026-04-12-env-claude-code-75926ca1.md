### この回の要点（先に読む）

- **変わったこと**: Claude Code がコマンドを実行するとき、「Mac本体で直接やる」か「隔離された箱（Docker）の中でやる」かを、作業内容に応じて自動で選ぶようになった。
- **いままでとの違い**: 以前は区別なくMac本体で実行していた。どんな操作でも同じ環境で動いていた。
- **メリット**: リスクのある操作（知らないプログラムのインストールなど）が隔離環境で実行されるため、Mac本体が汚染されない。

### 何が変わったか

#### 🏗️ AIが「安全な操作」と「危険な操作」を区別するようになった

Claude Code はファイルを編集するだけでなく、ターミナルでコマンドを実行できます。これは便利ですが、操作によってはリスクが異なります。

**安全な操作**（Mac本体で直接実行）:
- ファイルの編集・読み込み
- `npm run dev` / `npm run test` などの開発コマンド
- `git` 操作

**リスクのある操作**（Docker の隔離環境で実行）:
- `npm install <パッケージ>` — インターネットからプログラムをダウンロードする
- 外部サービスへの書き込み
- `rm -rf` などの削除操作
- 正体不明のスクリプトの実行

#### 📦 サンドボックス（Docker）とは何か

「砂場」の意味で、何をやっても外に影響が出ない使い捨ての実行環境です。

```
Mac本体（大切な資産）
└── Docker コンテナ（使い捨ての箱）
        └── ここで危険な操作を実行 → 箱ごと廃棄
```

終わったら箱ごと捨てるため、Mac本体は汚染されません。

#### 📋 CLAUDE.md にルールを書いて自動化した

「迷ったらサンドボックスを選ぶ」を原則として、操作カテゴリごとの判定テーブルを `CLAUDE.md`（Claude Code が毎セッション読む作業規約ファイル）に追記しました。これにより、毎回口頭で指示しなくても Claude Code が自動で環境を選ぶようになりました。

#### 🔑 「隔離環境」には2種類ある

この仕組みを正確に理解するために、「サンドボックス」の使い方が2種類あることを整理しておきます。

**① 通常起動（`claude`）＋ 危険な操作だけ箱に入れる**

普段は `claude` コマンドでローカルに起動します。Claude Code は Mac本体の上で動いていますが、リスクのあるコマンドを実行するときだけ、自動的に Docker の箱の中で実行します。

```
あなたのMac
┌─────────────────────────────────────┐
│                                     │
│  Claude Code（ローカルで動いている）  │
│                                     │
│  「npm install して」と言われた      │
│              ↓                      │
│  ┌────────────────────┐             │
│  │ docker の箱（一時）  │ ← その命令だけ箱の中で実行
│  │  npm install       │             │
│  └────────────────────┘             │
│  箱は終わったら即廃棄                │
│                                     │
│  ※ ~/.ssh/ や ~/.aws/ は見えている  │
└─────────────────────────────────────┘
```

git push やデプロイなど外部通信が必要な作業も問題なくできます。**日常の開発作業はこちら。**

**② 最初から `sbx run claude` で起動する**

Claude Code セッション全体を箱に入れて起動します。Mac本体のSSHキーやパスワードファイルに一切触れられないため、最大限に安全です。ただし git push やデプロイなど外部との通信が制限されるため、**コードを書くだけ・検証するだけ**の作業向けです。

```
あなたのMac
┌─────────────────────────────────────┐
│                                     │
│  ┌──────────────────────────────┐   │
│  │ サンドボックス（箱の中）       │   │
│  │                              │   │
│  │  Claude Code                 │   │
│  │                              │   │
│  │  ~/.ssh/ → 見えない ✅       │   │
│  │  ~/.aws/ → 見えない ✅       │   │
│  │  git push → ❌ できない      │   │
│  │  vercel deploy → ❌ できない │   │
│  └──────────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
```

**③ 使い分けの目安**

| 作業内容 | コマンド |
|---------|---------|
| 普段の開発（コード・デプロイ・git） | `claude` |
| 外部から持ってきた未知のコードを試す | `sbx run claude` |
| 医療データ・個人情報を扱う | `sbx run claude` |

①と②の判断はユーザーが起動前に決める必要があります。Claude Code 自身が「今すぐ sbx に切り替えて」と促すことはできません（すでにローカルで起動してしまっているため）。

MyHomePage の開発はコード編集・ビルド・デプロイがセットなので、**`claude`（ローカル）一択**です。CLAUDE.md の判定ルールが危険な操作だけ自動で箱に入れてくれるため、それで十分な安全性があります。

### 今回対応したこと

- ✅ `CLAUDE.md` に実行環境ルーティング（ローカル vs SBX）の判定テーブルを追加
- ✅ Docker コマンドの実行形式を明記（`docker run --rm -v $(pwd):/app ...`）
- ✅ 「迷ったら SBX を選ぶ」をデフォルトルールとして設定（分類外の操作への安全弁）
