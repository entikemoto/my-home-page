### この回の要点（先に読む）

- **変わったこと**: `/new-project`（新規プロジェクト作成コマンド）で新しいプロジェクトを作ると、`.claude/settings.local.json` が自動で置かれて GitHub に勝手に書き込めないよう最初からロックがかかる
- **いままでとの違い**: これまでは新プロジェクトを作った直後は「誰でも書込系 MCP を呼べる丸腰」状態。個別に deny を書き足すまで無防備だった
- **メリット**: 新規プロジェクトは初日から「勝手に PR を作る／Issue を立てる／リポジトリを作る」といった AI の暴発がブロックされる。PHI（患者情報）や confidential（事業秘密・個人情報）を指定した場合は Gmail や画面操作系もまとめてブロック

### 具体的に何ができるようになったか

- `python3 _tools/init_project.py --name X --type development --data-class PHI` のように data-class（機密度の分類）を指定できるようになった
- 生成物: `.claude/settings.local.json`（権限 600）と、値を埋めた `CLAUDE.md`（data-class の値と権限ハードニング説明が自動挿入）
- ベースライン（internal/public）: GitHub 書込系 MCP 12 種を deny（禁止）
- 強化版（confidential/PHI）: 上記 + Gmail 書込系 6 種 + computer-use 書込系 11 種を deny（禁止）

### なぜ必要だったか

- 2026-04-19 の監査レポート §5 で「新規プロジェクトテンプレート硬化」が最後の未対応項目だった
- グローバル設定の deny は後から書き足しているが、個別プロジェクトが丸腰だと「このプロジェクトで例外的に許可したい」運用が難しい（全体で許すか／個別で締めるかの二択になる）
- テンプレート段階で「閉じておいて開けていく」運用に寄せる方が、うっかり事故が起きにくい

### 動作確認

- `--data-class internal` と `--data-class PHI` の2パターンで実生成し、`.claude/settings.local.json` の中身と CLAUDE.md の data-class 値を目視確認 → OK → 削除
