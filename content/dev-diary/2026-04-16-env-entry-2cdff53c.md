### この回の要点（先に読む）

- **変わったこと**: Claude Code・Codex CLI・Gemini CLI の最新状況を確認し、開発環境に3つの改善を加えた。
- **いままでとの違い**: セッションの引き継ぎ失敗リスクがあり、セキュリティチェックも手動だった。
- **メリット**: 大事なメモが消えにくくなり、セキュリティチェックが自動になった。

---

### 何が変わったか

#### 1. 「保存してからコンパクション」が確実になった

AIとの会話が長くなると、Claude Code は自動的に会話を圧縮（コンパクション）します。これまでは「保存 → 圧縮」の順番のつもりが、稀に「保存が終わる前に圧縮が始まる」ことがありました。

今回、保存が失敗したら圧縮を止める仕組みを追加しました。

```
以前:  保存を試みる → （まれに保存が終わらぬまま）→ 圧縮スタート
今後:  保存が完了する → 圧縮スタート
        保存が失敗する → 圧縮をブロック（LAST_SESSION を守る）
```

大事な引き継ぎメモが飛ぶリスクがなくなりました。

#### 2. セキュリティチェックが自動になった

これまでは、コードを書いた後にセキュリティの問題がないか確認するのは「忘れなければ手動で行う」作業でした。

Claude Code v2.1.108 から、AIが Skill ツールを通じてビルトインのスラッシュコマンドを呼び出せるようになりました。これを活かして、**認証・データベース・外部APIなど、セキュリティに関わるコードを変更したとき、AIが自動で `/security-review` を実行する**ルールを追加しました。

認証・データベース・外部APIなど、影響の大きいコード変更でのうっかりミスを防ぐ安全弁になります。

#### 3. スマホ通知の準備が整った

Claude Code v2.1.110 から、長時間かかる作業が終わったときにスマートフォンへ通知を送る機能（Push Notification）が追加されました。

たとえば、長時間かかるビルド・データ処理・監視タスクを裏で走らせているとき、完了まで数時間待つことがあります。今後は「作業が終わったらスマホに通知」できるようになります（設定は Claude.ai の Settings → Remote Control から）。
