### この回の要点（先に読む）

- **変わったこと**: セキュリティ防御を8層追加し、ツールの最新化・スキル活用・事故対応を自動化する仕組みを3本柱で構築した
- **いままでとの違い**: セキュリティは手動チェック頼み、新しいツールの発見は気づいた人が調べ、スキルの使いどころは覚えている人だけが活用していた
- **メリット**: 「最新のツールを見逃さない」「セキュリティの穴を放置しない」「便利な機能を忘れない」が仕組みで保証されるようになった

### 何が変わったか

今回やったことは大きく3つの柱に分かれます。

```
┌─────────────────────────────────────────────────────┐
│           開発環境の「自律進化」3本柱                    │
├─────────────────────────────────────────────────────┤
│                                                     │
│  柱① 週次パイプライン                                 │
│  「新しいツールやスキルが出たら、自動で見つけてくる」      │
│                                                     │
│  柱② セキュリティ防御＋事故対応                         │
│  「守りを固め、万が一のときの手順書を用意する」            │
│                                                     │
│  柱③ スキル自動提案                                   │
│  「こんなときはこのスキルが使えますよ」とAIが教えてくれる」 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### 柱①「新しいツールを自動で見つけてくる」仕組み

開発の世界では、毎週のように新しいツールや便利な機能が公開されます。いままでは「自分で調べて、良さそうなら取り入れる」という手動作業でした。

今回、**毎週月曜の朝に自動で3つのチェックを走らせる仕組み**を作りました。

```
毎週月曜 AM 8:00（Mac が自動実行）
  ↓
  ✓ 今使っているツールに新バージョンは出てないか？
  ✓ 世の中で話題のスキル（便利な拡張機能）はないか？
  ✓ Claude Code 自体に新機能が追加されてないか？
  ↓
結果を「受信トレイ」に届ける → 人間が採否を判断
```

ポイントは「勝手に導入しない」こと。見つけてくるところまでが自動で、**実際に取り入れるかどうかは人間が判断**します。判断のための採点基準も文書化しました（4つの観点で点数をつけて、合計点で ✅採用 / ⏭️保留 / ❌不採用 を決める）。

#### 柱②「守りを固め、事故に備える」セキュリティ強化

セキュリティは「いたちごっこ」です。攻撃手法は日々進化するので、こちらも進化し続ける必要があります。今回、**8つの防御層**を一気に追加しました。

**〈自動で検出する仕組み〉**

- **パスワード漏洩防止**: コードを保存するときに、パスワードやAPIキー（外部サービスへの認証情報）がうっかり混ざっていないかを自動チェック。混ざっていたらコードの保存をブロックする
- **脆弱なライブラリの検出**: プログラムが使っている部品（ライブラリ）に「既知のセキュリティ問題」が見つかっていないかを自動スキャン
- **なりすまし命令の防御**: メールやWebページの内容をAIに渡したとき、その中に「この指示に従え」という悪意ある命令が埋め込まれていないかを自動検出。AIは外部から来た内容を「データ」として扱い、「命令」としては絶対に従わない
- **コード品質の静的解析**: Semgrep という専門ツールを接続し、OWASPトップ10（世界で最も多いセキュリティ問題のランキング）に該当するコードパターンを自動検出
- **GitHub連携**: GitHubというコード管理サービスと直接つないで、コードの変更履歴やプルリクエスト（変更の承認依頼）をAIが直接確認・操作できるようにした

ここで出てきた「Semgrep」と「GitHub」は、いずれも**外部の専門サービスとAIをつなぐ接続**です。AIが直接これらのサービスの機能を呼び出せるようになり、「人間がブラウザを開いて確認する」手間が省けます。このような外部サービスとの接続口を、技術的にはMCP（Model Context Protocol）と呼びます。いわば**AIが使える電話帳に新しい連絡先を追加した**ようなものです。

**〈事故が起きたときの手順書〉**

どれだけ守りを固めても、事故がゼロになることはありません。大事なのは**事故が起きたとき、最初の60分で何をするか**が決まっていること。今回、5つのケースについて手順書を作りました。

- パスワードが漏れた場合 → 即座に無効化 → 履歴から削除 → 影響範囲を調査
- 使っている部品に脆弱性が見つかった場合 → 危険度で優先順位をつけて修正
- AIへのなりすまし命令を検出した場合 → 無視 → 攻撃元を特定 → 防御パターンを更新
- 患者データへの不正アクセスの兆候 → 即座に遮断 → 法的報告義務を確認
- 外部サービスへの不審な通信 → 接続を切断 → 認証情報を無効化

**〈データの重要度タグ〉**

すべてのプロジェクトに「このプロジェクトが扱うデータはどれくらい重要か」というラベルを付けました。

- **PHI（最重要）**: 患者データ → 外部に絶対送らない、ローカル処理のみ
- **confidential（機密）**: メール内容、投資データ、事業情報 → 暗号化保存、共有制限
- **internal（社内）**: ニュース収集など → 公開はしないが厳格な保護は不要
- **public（公開）**: ホームページなど → 制限なし

AIはこのラベルを見て、データの扱い方を自動で変えます。たとえばPHIラベルのプロジェクトでは、外部にデータを送る操作はすべてブロックされます。

#### 柱③「スキルの使いどころをAIが教えてくれる」仕組み

この開発環境には現在23個のスキル（便利なショートカット機能）が入っています。でも「こんなときにはこのスキルが使える」と全部覚えるのは無理です。

今回、**AIへの指示文を分析して、関連するスキルを自動で提案する仕組み**を作りました。

```
あなた: 「セキュリティを見直したい」
  ↓  （AIが裏で23個のスキルと照合）
  ↓
AI: 「/security-audit が使えそうです（スコア: 8点）」
```

提案されても使うかどうかは自由。「今は要らない」なら無視すれば、AIは通常通り対応します。

#### 週次セキュリティ監査

毎週金曜 18:00 に、全プロジェクトを横断してセキュリティスキャンを自動実行します。パスワード漏洩・脆弱なライブラリ・コード品質の問題を検出し、レポートを生成します。重大な問題が見つかった場合はMacの通知で即座にお知らせします。

### 今回対応したこと

- ✅ パスワード漏洩防止フック（gitleaks）を追加
- ✅ 脆弱ライブラリ検出フック（pip-audit）を追加
- ✅ なりすまし命令防御フック（external-content-marker）を追加
- ✅ VoiceClinical プロジェクトに最厳格セキュリティ設定を適用
- ✅ Semgrep（静的解析）をAI接続に追加
- ✅ GitHub をAI接続に追加
- ✅ `/security-audit` スキル（全プロジェクト横断セキュリティ監査）を作成
- ✅ `/threat-model` スキル（脅威モデリング）を作成
- ✅ スキル自動提案フック（skill-router）を作成・23スキルをインデックス化
- ✅ 週次採用パイプライン（毎週月曜にツール最新化チェックを自動実行）を構築
- ✅ 採用判断の4軸評価基準を文書化
- ✅ セキュリティ事故対応プレイブック（5ケース）を作成
- ✅ 全10プロジェクトにデータ分類タグ（PHI/confidential/internal/public）を付与
- ✅ 週次自動実行スケジュール3件をMacに登録（毎週月曜: ツール最新化、毎週金曜: セキュリティ監査）
