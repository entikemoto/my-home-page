# 多言語対応（i18n）設計メモ

作成: 2026-04-14 | ステータス: 検討中

---

## 基本方針

| 項目 | 決定内容 |
|------|---------|
| URL構造 | `/en/` プレフィックス方式（SEO必須） |
| 切り替え | ページ上のボタンで言語切り替え |
| 翻訳方式 | Claude API（Sonnet）ビルド時一括翻訳 |
| キャッシュ | ハッシュベース差分翻訳（変更記事のみ再翻訳） |
| Fallback | 翻訳未対応記事は日本語版にリダイレクト |

---

## 翻訳パイプライン設計（OpenAI / Anthropic 視点）

### 2パス翻訳
1. **第1パス（Sonnet）**: 草稿翻訳
   - システムプロンプトに著者ペルソナ・用語集・スタイルガイドを渡す
   - 記事をタイトル・要約・本文・タグに分けて翻訳（精度向上）
2. **第2パス（Sonnet）**: 品質レビュー
   - 「日本語原文と英訳を比較し、意味の脱落・過剰翻訳・不自然な表現を修正」

### 翻訳プロンプトに明示する制約
- 原文にない情報を追加しない
- 数字・固有名詞・URL はそのまま保持
- 医療・投資アドバイスと誤認される表現を避ける
- Prompt Caching を活用（用語集・サイト概要をキャッシュ）

### コスト試算
- 現在の記事数 × 平均2,000トークン = 初回数百円程度
- 以後: 新記事1本あたり数円

---

## SEO設計（Google 視点）

- `hreflang` タグを全ページに設置
- 言語別サイトマップ（`sitemap-ja.xml` / `sitemap-en.xml`）
- ボタン切り替えだけでは英語コンテンツとしてインデックスされない → URLが必須

## 実装フェーズ

```
Phase 1（1〜2日）: URL構造・ルーティング・言語切り替えボタン
Phase 2（半日）:   UIテキスト・静的ページの英訳
Phase 3（1日）:    記事翻訳パイプライン構築（キャッシュ・2パス）
Phase 4（半日）:   hreflang・サイトマップ・SEO仕上げ
```
