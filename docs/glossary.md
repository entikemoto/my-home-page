# 用語集（Glossary）

## ドメイン用語

| 用語 | 定義 |
|------|------|
| **Article（個別記事）** | 1本の AI ニュース解説をまとめた最小の保存・公開単位。HP の中心的なエンティティ |
| **Dispatch（配信グループ）** | 朝3本・夜3本の配信単位。Article のメタデータであり、正本の主キーや URL には使わない |
| **Morning（朝便）** | 午前中に配信する Article 群（最大3本） |
| **Evening（夜便）** | 夜間に配信する Article 群（最大3本） |
| **Category（カテゴリー）** | Article の大分類。`ai-news`, `medical-ai`, `column`, `profile` の4種（将来拡張可） |
| **Tag（タグ）** | Article に付与する自由キーワード。正規化なし |
| **sourceUrl** | Article の元となったニュース記事の URL。引用要件のため必須 |
| **summary（要約）** | 元ニュースの要点をまとめた短文（200字以内推奨） |
| **commentary（解説）** | 著者が元ニュースに加えた考察・解説。MDX 形式で記述する |
| **draft** | 非公開フラグ。`true` の場合、本番ビルドでは記事を除外する |
| **canonical** | SEO の重複コンテンツ対策。HP を正本とし、note 等の重複掲載ページに HP URL を canonical として設定する |

## 外部サービス・連携ツール

| 用語 | 説明 |
|------|------|
| **CortexFlow2.0** | AI ニュース解説記事の生成ツール（別プロジェクト: `20260222_CortexFlow2.0`）。個別記事データを MDX 等の形式で HP に渡す |
| **Pagefind** | 静的サイト向けの全文検索ライブラリ。ビルド時にインデックスを生成し、ランタイムサーバー不要 |
| **note** | 外部ブログプラットフォーム。朝便・夜便のまとめ記事の配信場所。HP の役割とは分離している |

## 技術用語

| 用語 | 説明 |
|------|------|
| **SSG（Static Site Generation）** | ビルド時に全ページを静的 HTML として生成する方式。本サービスの基本方針 |
| **App Router** | Next.js 13 以降の新しいルーティング方式（`app/` ディレクトリ）。本プロジェクトはこれを使用する |
| **MDX** | Markdown の中に React コンポーネントを埋め込める拡張フォーマット。commentary の記述に使用 |
| **OGP（Open Graph Protocol）** | SNS シェア時にタイトル・画像・説明を表示するためのメタタグ。各記事ページに設定する |
| **LCP（Largest Contentful Paint）** | ページの主要コンテンツが表示されるまでの時間。2.5 秒未満を目標にする |
| **ArticleMeta** | Article から `commentary`（本文）を除いた型。記事一覧表示に使用し、転送データ量を削減する |
