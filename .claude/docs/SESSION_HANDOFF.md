# セッション引き継ぎ

## 目的

長時間セッションや、翌日に作業を再開する際のコンテキスト喪失を防ぐ。

---

## セッション終了時のチェックリスト

セッションを終える前に、以下を更新してください。

### 1. 作業状態の記録

```markdown
## 最終更新: {YYYY-MM-DD HH:MM}

### 今日やったこと
-

### 途中の作業（未完了）
- [ ]

### 次にやるべきこと（優先順）
1.
2.
3.

### ブロッカー（あれば）
-

### 重要な決定事項
-

### 参照すべきファイル
-
```

### 2. 更新すべきファイル

| ファイル | 更新内容 |
|----------|----------|
| `docs/project-status.md` | Phase、Next Actions、Blockers |
| `.claude/docs/SESSION_HANDOFF.md` | この引き継ぎ情報 |
| `docs/DESIGN.md` | 設計決定があれば |

---

## セッション開始時のプロンプト

翌日や別セッションで再開する際に、以下を貼り付けてください。

```markdown
あなたはこのプロジェクトの開発エージェントです。
前回のセッションから引き継ぎます。

1. まず以下を読んで、現状を把握してください：
   - `CLAUDE.md`
   - `docs/project-status.md`
   - `.claude/docs/SESSION_HANDOFF.md`

2. 前回の「途中の作業」と「次にやるべきこと」を確認し、要約してください。

3. ブロッカーがあれば、解決策を提案してください。

4. 準備ができたら「続きを始めます」と言ってください。
```

---

## 自動保存のタイミング

以下のタイミングで SESSION_HANDOFF.md を更新することを推奨：

- [ ] 1時間ごと（長時間作業時）
- [ ] 大きなタスクが完了したとき
- [ ] エラーや問題に遭遇したとき
- [ ] セッションを終了する前（必須）

---

## コンテキスト節約のコツ

1. **大きな出力は要約**: Gemini のリサーチ結果は `.claude/docs/research/` に保存し、要約だけ参照
2. **履歴のクリア**: 不要になった議論は `/clear` でリセット
3. **ファイルで受け渡し**: 長い指示はファイルに書いて `@file` で参照
